# 实战模拟室

养成材料副本（基础材料 / 代理人晋升等），消耗电池。从快捷手册-训练-实战模拟室进入。

## 何时出现 + 状态流转

快捷手册传送 → 副本选择 → 出战 → 战斗（借「战斗画面」battle）→ 战斗结果（完成，借「战斗画面」battle）→ **战斗结束-获得奖励** → 回副本选择。

**子态**（同 screen `实战模拟室`，多子态共用 screen_info `combat_simulation.yml`）：

| 子态 | 入口 | 出口 |
|---|---|---|
| 副本选择（挑战等级） | 传送进 / 获得奖励返回 | 选副本类型→下一步 / 直接出战 |
| 出战 | 副本选择点下一步 | 点出战→战斗（战斗画面） |
| 战斗结束-获得奖励 | 战斗结果点「完成」后弹出 | 点确认→回副本选择 / 点前往→物品获取途径 |

> 战斗中（攻击按钮）/ 战斗结果（完成·再来一次）归通用「战斗画面」(battle.yml)，不在本 screen。**战斗结束画面按玩法分**：实战模拟室=获得弹窗（本 screen）/ 防卫战=挑战结果(shiyu_defense.yml) / 其他玩法各自 screen。

## 识别特征（稳定锚点）

- **副本选择态**：`挑战等级`（右上）+ `下一步`/`出战`。
- **出战态**：`出战`/`预备出战`/`确定并出战` + 角色编队。
- **战斗结束-获得奖励**：`战斗结果-获得`（「获得」标题）+ `战斗结果-获得-前往`/`战斗结果-获得-确认`。
- 无 `id_mark: true` → `is_precise=false`。

## 识别快照

### 1. 战斗结束-获得奖励(source_image: screens/实战模拟室/战斗结果-获得奖励.webp)

战斗胜利、点「战斗结果-完成」之后弹出的获得奖励弹窗（实战模拟室特有；防卫战/空洞等玩法结算不同，各自 screen）。普通攻击按钮已消失（战斗 UI 收起），背景为战斗场景（带 lore 文字）。显示获得的材料（如黄金汽水瓶盖）。

- **匹配画面**：`实战模拟室` `is_precise=false`（3 个 area 命中，得分最高，压过恢复电量/咖啡店等的「确认」误匹配）
- **匹配 area**：

  | area_name | 类型 | conf |
  |---|---|---|
  | 战斗结果-获得 | text | 0.997 |
  | 战斗结果-获得-确认 | text | 0.9998 |
  | 战斗结果-获得-前往 | text | 0.9985 |

- **全量 OCR**：`获得`、`黄金汽水瓶盖`、`前往`、`确认`（+ 背景 lore：`AN ABNORMAL`/`THE INTERNAL SPA`）。
- **布局**：居中黑色半透明弹窗 ——「获得」标题 + 金色瓶盖图标 + 数量 + 物品名 + 底部左「前往」（黄箭头）/右「确认」（绿对勾）。

> 该子态不被 `NormalBattleOp`/`CombatSimulation` 主动处理（它们到 `ChooseNextOrFinishAfterBattle` 点完成即结束）；弹窗「确认」由游戏流程或后续 op 收尾。建档是为了让框架识别该画面（此前因仅「确认」文字命中，被误判成恢复电量/咖啡店等）。
