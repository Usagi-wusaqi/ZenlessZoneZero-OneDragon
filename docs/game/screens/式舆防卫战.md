# 式舆防卫战

绝区零周期战斗玩法(剧变节点 14 天刷新;稳定/纷争节点永久)。app `shiyu_defense`。入口:大世界 → 快捷手册(作战 tab)→ 式舆防卫战「前往」→ 传送确认 → 选关主界面。

## 何时出现 + 状态流转

传送进 → 选关主界面 → [选节点] → 弱点/抗性 + 选配队 → 出战 → 战斗 → 下一节点 / 战斗结束 → 全完成 → 领奖 → 返回大世界。

**子态**(同 screen `式舆防卫战`,多子态共用 screen_info `shiyu_defense.yml`):

| 子态 | 入口 | 出口 |
|---|---|---|
| 选关主界面 | 传送进 / 战斗结束返回 | 点节点-XX → 弱点 |
| 弱点+抗性+选配队 | 选关后 | 点角色头像 → 选配队 |
| 预备编队管理+选队 | 点角色头像 | 选队(+200px)+预备出战 → 出战 |
| 战斗(含阶段内移动) | 出战(Deploy) | `ShiyuDefenseBattle` op 跑完该阶段 → 下一阶段 / 结算 |
| 战斗后移动(阶段内层切换) | 一层敌人清完 | op 自移动+F 交互 → 下一层 / 该阶段胜利 |
| 挑战结果(结算) | 防线/阶段胜利 | 战斗数据/退出 → 下一防线或选关 |
| 领奖 | 全节点完成 | 奖励入口 → 全部领取 → 关闭 → 返回 |

**多间模式**(节点 5):点节点-05 → `式舆防卫战-三间选择`(选 3 间 → 各自战斗,见 `shiyu_defense_select_3.yml`)。

**剧变节点进度**:`剧变节点进度` area 显示 X/Y(如 3/5)。⚠️ **OCR 常连写为 `315`**(`/` 被识成 `1`),app `choose_node_idx` 用首尾数字法解析(`current=digits[0]`, `total=digits[-1]`,见 PR #2510 / issue #2506)。

## 选配队与触发战斗(本次实操,2026-07-19)

**选配队**(app `ChoosePredefinedTeam` + `calc_teams`):
1. 选关后进弱点/编队界面(关卡特性 / 推荐属性=弱点 / 强敌抗性 / 挑战目标 / 2 阶段 TEAM SELECT)。
2. 点「预备编队」按钮(413,1012)→ 预备编队管理列表(显示 `team_config` 全部编队 + 角色 + 各自 weakness)。
3. ⚠️ **选队 = 点编队名右侧 +200px**(`ocr_result.max.center + Point(200, 0)`,见 `choose_predefined_team.py:82`)—— 点名称本身是**改名弹窗**,只有 +200px 才是选中。这是关键坑。
4. `calc_teams` 配置驱动:OCR 弱点 area → 匹配预备编队 `weakness_list`(`ShiyuDefenseConfig`)→ DFS 选最优(弱点+1 / 抗性-1 / 避代理人头像冲突 `is_team_conflict`)。本次阶段1 风·物理抗火 → team 1 简薇耀(PHYSICAL);阶段2 火·物理抗冰 → team 4 火莱凯露(FIRE);避开 team 6 简赛露(与 team 4 共 `caesar_king` 冲突)。
5. 选 2 阶段(01/02 TEAM 标记)+「预备出战」→ 回配队选择(显示选的编队)+「出战」。

**触发战斗**(app `deploy` → `shiyu_battle` → `ShiyuDefenseBattle` op):
1. 点「出战」→ 战斗画面(角色离敌人**有距离**,HP 满,未触发战斗)。
2. **`ShiyuDefenseBattle` op 内部完整封装移动+战斗+阶段切换**(不需外部干预,op 自带):
   - `wait_battle_screen`:等 `战斗画面.按键-普通攻击` → `start_move`:检测**防卫战倒计时**(pipeline,4 位数字)→ 有倒计时 = 在战斗 → `start_auto_battle`(跑配队 auto_battle 脚本);无倒计时则检测**距离**(`check_battle_distance`)→ `move_w`(press_time = 距离/7.2)靠近,距离偏左/右则 `turn_by_distance` 转向。
   - `auto_battle`:战斗中每秒检测倒计时,**连续 5 秒无倒计时** = 该层敌人清完 → `STATUS_NEED_SPECIAL_MOVE`。
   - `move_after_battle`(**阶段内层切换,op 自处理**):检测倒计时(重现=返回战斗重启 auto_battle)/ `战斗画面.按键-交互`(命中则 `interact` F,press_time=0.2,进下一层)/ 检测距离/传送点(`防卫战空洞传送点` pipeline)→ 自 `move_w` 靠近 → 直到无交互按钮无攻击按钮 = `STATUS_TO_NEXT_PHASE`(该弱点阶段胜利)。
3. **节点多弱点阶段**(2 阶段):app `shiyu_battle` 每阶段调一次 `ShiyuDefenseBattle(phase_team_list[phase_idx].team_idx)`,phase_idx 递增;阶段全胜 → `STATUS_NEXT_NODE` → `to_next_node`。
4. ⚠️ **不要绕过 op 手动 `key_tap w` / 手动 `interact`**:「距离+入口」是 op 的 `move_after_battle` 内部状态,op 自己移动自己 F;外部操作会干扰 op(实测:手动 run `ShiyuDefenseBattle` 时画面若已在阶段切换中,op 仍会自处理,外部瞎点反而乱)。

### 4. 战斗画面(source_image: screens/式舆防卫战/战斗画面.webp)
HP / 敌人名(UPROAR)/ 攻击按钮模板(战斗态)。进战斗时角色离敌人有距离,**`ShiyuDefenseBattle` op 的 `start_move` 自动按距离 `move_w` 靠近 + 检测倒计时触发 auto_battle**(不需手动移动)。

### 5. 一层结束 → 战斗后移动(阶段内层切换,source_image: screens/式舆防卫战/一层结束-阶段切换.webp)
一层敌人清完(连续 5 秒无倒计时)→ HP 恢复满(28393,新层等级)+ **距离显示(`14.9m`→`6.7m`,逐渐接近)**+ **谜团入口(下一层入口,图形,OCR/vision 难认)**。这是 **`ShiyuDefenseBattle` op 的 `move_after_battle` 节点内部状态**:op 自检测距离 / 传送点(`防卫战空洞传送点` pipeline)/ 交互按钮 → 自 `move_w` + `interact` F 进下一层 → 倒计时重现 = 返回 auto_battle。**op 完整封装,不需外部干预**(早前误记"无 op 暴露结束信号"是错的,已据 `shiyu_defense_battle.py` 纠正)。

### 6. 入口前可交互(source_image: screens/式舆防卫战/入口前可交互.webp)
移动到入口前 → **`战斗画面.按键-交互` template 命中(conf 0.94)** + OCR `F`(1438,1018)= F 可交互。这是 op `move_after_battle` 检测到交互按钮后的 `interact` F(press_time=0.2)触发点 → 进下一层。**客观识别靠"按键-交互" template 命中**,不靠 vision 推断(vision 难认入口结构)。
