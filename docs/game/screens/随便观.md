# 随便观(经营玩法)

澄辉坪「随便观」经营玩法:周期托管 + 多子玩法(游历 / 饮茶仙 / 制造坊 / 售卖铺 / 好物铺 / 邦巢 / 德丰大押)。**重 app** —— 主 app `SuibianTempleApp` 调度 7+ 子 op,每子 op「从入口进 → 操作 → 返回入口」。本文基于**实拍截图 + screen_info + 代码**三层(截图归档测试仓 `screens/随便观/`)。

## 入口 + 状态流转(实拍确认)

**传送「随便观」直达内部**(2026-07-18 实拍):地图传送点「随便观」(澄辉坪区域)传送后**直达随便观内部**(自动托管/入口画面),不落大世界。

> ✅ **传送行为(2026-07-18 实拍 2 次)**:传送「随便观」**直达随便观内部** —— 第1次(托管中)→自动托管画面;第2次(托管停)→入口手动态,**两次都直达内部,不落大世界 interact 狮耶**。interact 狮耶 = 从大世界狮耶旁 interact 进(另一进法,不经传送);app `goto_suibian_temple` 的 OCR「前往随便观」= **兜底**(代码确认 `suibian_temple_app.py:72-81`:节点先 OCR 点「前往随便观」,点不到则检测「随便观-入口」success —— 只在传送落大世界时命中,直达内部时不走)。

> ✅ 入口是 **interact 狮耶**(传送点旁 NPC,`>`名字`<` 可交互),**不是** OCR「前往随便观」文字。app `goto_suibian_temple` 的 `round_by_ocr_and_click_by_priority(['前往随便观',...])` 点的是 interact 狮耶后屏幕出现的交互提示文字。Transport 后传送点就在狮耶交互范围内(**app 无 move,不走**;走了反而过头错过交互点)。

入口两态:
- **自动托管中态**(入口-自动托管中.webp):中央「LV.45 / 完成复兴 / 自动托管中 / 可关闭自动托管进行手动操作」;游历/经营 tab **隐藏**(锁)。
- **手动态**(入口-手动态.webp):托管停止后,「游历」+「可派遣」/「经营」+「可制造」tab 显示 + 「暂停营业」。

**自动托管中 → 停止托管路径**(实拍):入口左下「托管中」(`129,967`,左半屏)→ 自动托管/经营日志画面 → 「停止托管」(`1159,879`)→ 确认弹窗(警告不足时间无收益)→ 获得奖励(百通宝/云纹徽结算)→ 入口手动态。

app 流程分两条(`config.auto_manage_enabled`,托管与子玩法**互斥** —— 托管中入口 tab 锁,没法手动派遣/制造):
- **开(默认)**:识别初始 → (传送 → interact 狮耶) → `handle_auto_manage`(`SuibianTempleAutoManage`:停止托管领收益 → **click「开始托管」重启托管**)→ 返回大世界。**不跑子玩法**(`handle_adventure_squad` 仅 `@node_from(..., status='未开启自动托管')`)—— 托管跑收益。
- **关**:识别初始 → (传送 → interact 狮耶) → `handle_auto_manage`(跳过,返 '未开启自动托管')→ 游历 → 饮茶仙(条件) → 饮茶仙后再游历 → 制造坊 → 售卖铺 → 好物铺(条件) → 邦巢(条件) → 德丰大押(代码未开启) → 返回大世界(手动子玩法)。
