# 邮件

## 何时出现 + 状态流转

从**菜单**底部「邮件」tab 进入(文本「邮件」@~(532,1026))。每日领取邮件附件的入口(`email` application)。

| 子态 | 入口(从哪来) | 出口(动作→下一态) |
|---|---|---|
| ① 列表态(默认) | 菜单点「邮件」tab | 全部领取(有可领)→ ②;全部领取(无可领)→ 停留;按钮-返回 → 菜单 |
| ② 确认弹窗(已验证,截图待补) | 列表态点「全部领取」且有可领附件 | 确认 → 列表态(已领取);点外层 → 列表态 |

> `BackToNormalWorld` 可从邮件经菜单返回大世界。

## 识别特征(稳定锚点)

- **标题「邮件」**(@左上 (178,31),`标题-邮件` area,`id_mark`)—— 识别本画面的主锚点。
- **按钮-返回**(模板 `menu/back` @左上 (92,23),goto 菜单)。
- **全部领取**(文字按钮 @左下 (154,1012),`id_mark`;**灰色也能 OCR 识别**,非可领状态判据)。
- **列表结构**(vision 核对):左侧邮件列表(每项 = 发件人 / 标题 / 剩余天数 / 已读绿勾,选中项黄框高亮)+ 右侧正文预览 + 右下附件格(道具图标或 `EMPTY`)。
- **状态指示**(OCR 看不见,需 vision):列表选中态(黄框)、已读标记(绿勾)、重要邮件星标(黄星)、附件可领(格内道具图标 + 黄勾)。
- **易变值**(勿当特征):邮件标题/发件人/剩余天数/正文/附件内容/数量 `8/1000` 全动态。

## 可交互元素

- **全部领取**(文字按钮):一键领取所有邮件附件。
- **删除已读**(文字按钮 @~(424,1012))。
- **删除邮件**(文字按钮 @~(1685,1012))。
- **邮件列表项**:点击选中查看正文(选中项黄框高亮)。
- **按钮-返回**(图形按钮,模板 `menu/back`,左上箭头):返回菜单。
- **状态图标**(OCR 难读,vision 可见):已读绿勾 / 重要星标 / 附件可领勾 —— 展示性,非常规点击目标。
