# 传送(大世界快速移动)

> 传送是**通用机制**(非玩法),跨多个 screen。本文集中传送机制;画面识别见 [地图](../screens/地图.md) / [3D地图](../screens/3D地图.md)。

## 两种地图

| | 普通地图 | 3D 地图 |
|---|---|---|
| 用途 | `MapTransport`(传送) | `world_patrol`/锄大地(立体探索) |
| 入口 | 大世界**按 `N`**(或点「地图」area template `map`) | 大世界**按 `M`** |
| 传送流程 | 区域 → 传送点 → **确认弹窗** | 区域 → 子区域 → 筛选 → 图标 → **前往**(无确认弹窗) |
| screen | [地图](../screens/地图.md) | [3D地图](../screens/3D地图.md) |

⚠️ **易混(实操踩坑)**:`M` 开的是 **3D 地图**,不是普通传送;**普通传送按 `N`**。`Transport.open_map` 点大世界「地图」area(template `map`)开普通地图。

## 传送点类型(2026-07-18 实拍)

- **instance 类**(随便观 / 饮茶仙 / 临界推演):**直达对应玩法内部**(不落大世界)。「随便观」2 次实拍都直达内部(第1次托管画面 / 第2次入口手动态)。
- **大世界 NPC 点**(阿朔 / 汀曼咖啡 / 录像店营业点):落大世界对应 NPC 旁。

## 传送行为(待持续验证)

- instance 传送**直达内部**(2 次实拍)。
- ⚠️ app `goto_suibian_temple` 的「interact 狮耶」逻辑:传送既直达,interact 狮耶可能是**冗余/兜底**(或特定条件落大世界 interact NPC)。待确认。
- interact 狮耶 = 从大世界狮耶旁 interact 进随便观(另一进法,不经传送)。

## 传送点解锁

传送点需**探索解锁**(未跑图 = 未解锁 = Transport 选不中 → 失败)。新版本玩法 / 新城区要先跑图解锁传送点。`run_operation Transport` 失败常见根因 = 目标点未解锁。

## 相关 op

- `Transport(area, tp)`(`src/zzz_od/operation/transport.py`):返回大世界 → 打开普通地图(点「地图」area)→ `MapTransport` → 等加载。
- `MapTransport`(`map_transport.py`):区域选择(整屏 OCR + 横向 drag)→ 选传送点(OCR 列表)→ 点确认。
- `TransportBy3dMap`(`world_patrol`):3D 地图传送(区域→子区域→筛选→图标→前往)。
