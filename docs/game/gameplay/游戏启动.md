# 游戏启动(开服到登录前的画面序列)

启动游戏后、进入登录前的固定**启动序列**(三张连续过渡页,均无操作、自动推进):

1. **[米哈游启动页](../screens/米哈游启动页.md)** — 米哈游品牌 + 合规展示(黑底 logo + 底部合规文字)
2. **[警告:游戏前详阅](../screens/警告_游戏前详阅.md)** — 光敏性癫痫警告
3. **[绝区零标题页](../screens/绝区零标题页.md)** — 游戏 logo(`绝·区·零`)
4. **[打开游戏](../screens/打开游戏.md)** — 登录页(自动登录:loading → ready,点 `点击进入游戏` 进入;手动登录多子态见 [登录游戏](登录游戏.md))

> `involves_screens` 列表顺序 = 实际出现顺序;正文用编号链展开。

## 自动化要点
- 三页都**无可点击元素**,`open_and_enter_game` 靠**等待 / 轮询超时**跳过,不能点击推进。
- 三页目前都是 **screen_info 缺口**:启动页 / 警告页会被模糊误匹配成「菜单」/「打开游戏」,标题页完全无匹配。补 screen_info 时用各自稳定特征(miHoYo logo / 警告标题 / `绝·区·零`)区分,避免误识别。

## 信息来源与置信度
- ✅ **实拍确认**:启动三页序列 + 自动推进(无操作,`open_and_enter_game` 等待/轮询超时跳过)。
- ℹ️ **screen_info 缺口**:三页无精准 screen_info(误匹配 / 无匹配),补时用稳定特征区分。
