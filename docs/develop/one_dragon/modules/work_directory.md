# 项目工作目录

## 目标

项目工作目录是配置、日志、资源、虚拟环境和调试文件的统一根目录。它必须在进程生命周期内保持稳定，不能随 `os.chdir()` 或模块导入顺序变化。

## 默认规则

`one_dragon.utils.os_utils.get_work_dir()` 按以下顺序返回目录：

1. 已通过 `set_work_dir()` 显式指定的目录；
2. PyInstaller 冻结运行时的可执行文件所在目录；
3. 源码运行时根据 `os_utils.py` 位置推导出的项目根目录。

`get_work_dir()` 不缓存当前进程目录，也不使用 `os.getcwd()` 作为冻结程序的默认根目录。因此，从快捷方式、命令行或其他程序启动时，调用方的当前目录不会影响项目文件位置。

## 显式覆盖

安装器需要由用户选择安装目录，因此在导入配置、日志、翻译等业务模块前调用：

```python
set_work_dir(work_dir: str | Path | None) -> None
```

传入路径后，后续工作目录查询均使用该目录；传入 `None` 时恢复默认规则。`os.chdir()` 仍可用于兼容依赖相对路径的外部命令，但不负责确定项目工作目录。

## 导入约束

普通启动器和 RuntimeLauncher 无需依赖导入顺序修正工作目录，它们在冻结环境中默认使用可执行文件所在目录。

安装器必须先完成目录选择和显式覆盖，再导入会在模块级计算路径的业务模块，避免模块级常量提前绑定到安装器所在目录。
