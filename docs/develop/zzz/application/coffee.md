# 咖啡应用运行流程

## 概述

咖啡应用（`CoffeeApp`）负责前往咖啡店、选择并点单每日咖啡。咖啡店本身不执行副本战斗；喝后挑战开启时，应用返回大世界后复用完整的体力计划应用。

## 配置项

- **传送地点**：
  - `六分街 - 咖啡店`：默认地点。传送后转向正西，向前移动并交互，进入咖啡列表点单。
  - `澄辉坪 - 汀曼咖啡`：传送落点正对入口，直接交互后通过汀曼大师的对话选项点单。
- **咖啡选择**：
  - `优先双倍活动`：喝后挑战采用“沿用体力计划”且开启双倍活动时选择`浓缩咖啡`，否则选择`汀曼特调`。
  - `汀曼特调`：只选择汀曼特调，也是默认选项。
  - `浓缩咖啡`：只选择浓缩咖啡。
- **喝后挑战**：
  - `沿用体力计划`：返回大世界后运行默认体力计划，按配置处理双倍活动，也是默认选项。
  - `沿用体力计划（不考虑双倍活动）`：运行体力计划，但本次执行跳过双倍活动检查。
  - `不挑战`：只喝咖啡，不运行体力计划。

咖啡不再单独配置卡片数量、预备编队和自动战斗，这些设置统一由体力计划维护。

## 主要流程

```
传送到咖啡店 → 交互并选择咖啡 → 点单/确认电量 → 返回大世界 → 运行体力计划（可选）
```

- 六分街咖啡店直接从咖啡列表选择；汀曼咖啡通过对话选项选择。
- 已喝过咖啡时直接回到大世界。
- 每次选择时会优先尝试尚未喝过的不占用上限咖啡；喝完后回到选择咖啡节点，再按配置选择普通咖啡。
- 不占用上限的咖啡使用专用确认分支；无增益咖啡不进入咖啡对应的挑战分支，但喝后挑战开启时仍会运行完整体力计划。

## 与体力计划的联动

喝后挑战开启时，咖啡应用获取当前账号、实例的默认组 `ChargePlanApp` 并执行，不再自行构造临时副本计划。选择“不考虑双倍活动”时，将 `ignore_double_reward` 设为本次执行标记；体力计划执行结束后会清除该标记并重置本轮计划的跳过状态，避免影响后续应用运行。

## 旧配置清理

`CoffeeConfig` 初始化时会删除旧版的挑战选项、咖啡独立副本配置及已废弃的“结束后运行体力计划”开关。清理发生后配置会立即保存；代码中的清理逻辑计划在 2026-10-12 后移除。
