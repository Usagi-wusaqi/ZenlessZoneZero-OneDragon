# 影像店营业(自动化方案)

> 本文档讲**怎么自动化**影像店营业玩法(脚本流程 / config / 设计决策)。
> **玩法机制**见 [gameplay/影像店营业.md](../../../game/gameplay/影像店营业.md);**画面识别**见 [screens/影像店营业.md](../../../game/screens/影像店营业.md);**文档分工**见 [harness/doc_organization.md](../../harness/doc_organization.md)。

**自动化目标**:每日完成录像店营业(选宣传员 + 上架录像带 + 开始营业);已营业则跳过。

## 1. 玩法与自动化的关系

玩法机制详见 [gameplay](../../../game/gameplay/影像店营业.md)。脚本覆盖的子玩法及其自动化角色:

| 子玩法 | 游戏含义(详见 gameplay) | 脚本角色 |
|---|---|---|
| 选宣传员 | 派驻代理人影响营业 | 按 `config.agent_name` 选(OCR 名称优先 + 头像兜底 + 滚动) |
| 上架录像带 | 为 3 主题上架匹配录像带 | 主题 OCR + difflib 匹配 16 主题 + 上架筛选循环 |
| 开始营业 | 确认营业赚取丁尼 | 点开始营业 → 确认 → 营业后确认 |
| 已营业跳过 | 正在营业态 | `check_running` 检测 → 关闭经营页面 → 返回 |

## 2. 脚本流程

主 app `RandomPlayApp` 节点链(详见代码 + [screens](../../../game/screens/影像店营业.md)):

1. `transport`(传送;失败重试 from「等待经营画面加载」)
2. `move_and_interact`(POINT_1 `turn_to_angle(0)`+`move_w` / POINT_2 直接 interact;sleep 1 + interact 0.2 防失效)
3. `wait_run`(等待经营画面加载;昨日账本关闭 / 经营状况点击 / 「查看经营状况」对话框)
4. `check_running`(正在营业→关闭经营页面;否则→选宣传员)
5. `choose_promoter`(已选「换下」→跳过;OCR 名称 + 头像匹配 + 滚动兜底 + 确认)
6. `check_video_theme`(识别 3 主题,difflib 匹配 16 主题)
7. `click_video_entry` → `check_recommended`(推荐上架→返回;上架筛选→上架)
8. `click_filter` → `choose_theme` → `choose_onshelf`(主题筛选 + 上架,3 主题循环)
9. `back` → `start`(开始营业)→ `confirm_business`(确认)→ `confirm`(营业后确认)
10. `back_to_world`(`BackToNormalWorld` 回大世界)

**设计决策(为什么)**:
- **POINT_2 推荐**:传送落点正对入口,直接 interact(POINT_1 需 turn+move,易出错)。
- **宣传员 OCR + 头像兜底**:名称 OCR 优先,失败用头像模板匹配(`predefined_team/avatar_`)。
- **主题 difflib**:16 主题 OCR + difflib 模糊匹配(主题名 OCR 可能不准)。
- **已营业跳过**:`check_running` 检测「正在营业」→ 关闭经营页面 → 返回(不重复营业)。

## 3. 支持配置(`RandomPlayConfig`)

- `transport_point`:传送点(POINT_1 录像店-柜台 / POINT_2 澄辉坪-录像店营业点)。
- `agent_name_1` / `agent_name_2`:宣传员(代理人名;`RANDOM` 随机选)。

## 4. 注意事项

- **每日首次**:昨日账本弹窗(`wait_run` 关闭后再判经营状况,防残影 / 未关闭)。
- **interact 防失效**:`move_and_interact` sleep 1 + interact press_time 0.2(issue #2405 / #2395 / #2328)。
- **跨画面 op**:`transport`(Transport)+ `back_to_world`(BackToNormalWorld)跨画面。
- **营业确认弹窗待补**:开始营业-确认 / 营业后确认(画面 fixture 待拍)。
