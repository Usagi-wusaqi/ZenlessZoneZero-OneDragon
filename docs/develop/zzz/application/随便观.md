# 随便观(自动化方案)

> 本文档讲**怎么自动化**随便观玩法(脚本流程 / 配置 / 前置 / 设计决策)。
> **玩法机制**(游戏里随便观是什么、资源、循环)见 [gameplay/随便观.md](../../../game/gameplay/随便观.md);**画面识别**见 [screens/随便观.md](../../../game/screens/随便观.md);**文档分工**见 [harness/doc_organization.md](../../harness/doc_organization.md)。

**自动化目标**:按随便观**满级(复兴度 45)后**的状态实现,尽量将收益最大化(手动经营 > 托管低保,见 gameplay「托管 vs 手动」)。

## 1. 玩法与自动化的关系

玩法机制详见 [gameplay](../../../game/gameplay/随便观.md)。本脚本覆盖的子玩法及其自动化角色:

| 子玩法 | 游戏含义(详见 gameplay) | 脚本角色 |
|---|---|---|
| 经营(制造 + 售卖) | 制造商品 → 售卖赚云纹徽 | **自动托管**(停托管领收益 → 重启托管) |
| 游历 | 派邦布产材料 | **自动托管**(托管不耗材料) |
| 饮茶仙 | 定期采办委托换百通宝 | 脚本只**提交**定期采办(不刷新,交托管完成);**邻里心愿是一次性固定任务,做完清空,不跑** |
| 好物铺 | 云纹徽换材料 / 插件 | 脚本按配置购买 |
| 邦巢 | 花百通宝雇邦布 | 脚本按配置购买(关快速选择) |
| 德丰大押 | ~~百通宝/云纹徽换周期奖励~~(2.5 移除,改德丰珍宝) | op 基于 2.4、**2.5 tab 移除失效**(死代码);app `handle_pawnshop` 占位直返"未开启"**短路**(`suibian_temple_app.py:175-176`)—— config `pawnshop_*_enabled` 默认 True 但 app 不调 op,失效 op 不会跑到,故无运行错误。待按德丰珍宝重写。 |

## 2. 脚本流程

主 app `SuibianTempleApp` 节点链(详见代码 + [screens](../../../game/screens/随便观.md)):

1. 识别初始画面 → 传送澄辉坪 + interact 狮耶 → 随便观-入口
2. `handle_auto_manage`:停止托管领收益 → **重启托管**(config `auto_manage_enabled` 开;关则跳过、走手动子玩法链)
3. (config 关托管时)游历 → 饮茶仙(条件)→ 再游历 → 制造坊 → 售卖铺 → 好物铺(条件)→ 邦巢(条件)→ 德丰大押(未接入)→ 返回大世界

**设计决策(为什么)**:
- **饮茶仙只提交不刷新**:定期采办委托类型不固定,手动刷新性价比低;委托交给自动托管完成。脚本只做「能立即提交的」。
- **托管优先(config 默认开)**:满级后托管低保已够,且不耗材料;手动子玩法链是 config 关时才跑(精细优化收益,见 gameplay)。
- **饮茶仙后再游历**:饮茶仙补料可能触发游历派遣需求,故饮茶仙后再跑一次游历。
- **德丰大押未接入**:op 已实现,但 app 层 `handle_pawnshop` 占位返「未开启」,待接入。

## 3. 支持配置(`SuibianTempleConfig`,对齐代码)

> 标 **[已实现]** / **[计划]**(代码现状)。字段见 `suibian_temple_config.py`。

- **自动托管**:`auto_manage_enabled`(开 → 只托管;关 → 跑手动子玩法链)[已实现]
- **饮茶仙**:`yum_cha_sin`(启用)、`yum_cha_sin_period_refresh`(定期采办是否刷新)[已实现]
- **游历**:`adventure_duration`(时长,默认 20h)、`adventure_mission_1~4`(副本,默认科研院旧址系列)[已实现]
- **制造坊**:`craft_drag_times`(最大下拉次数,跳过低级物品)[已实现]
- **好物铺**:`good_goods_purchase_enabled`(启用;当前只买邦布能源插件)[已实现] · TAB1~4 购买优先清单[计划]
- **邦巢**:`boo_box_purchase_enabled`(启用)、`boo_box_adventure_price` / `craft_price` / `sell_price`(各类型最低价 S1~S4)[已实现]
- **德丰大押**:`pawnshop_omnicoin_*`(百通宝)、`pawnshop_crest_*`(云纹徽)[代码已实现,app 未接入]

## 4. 注意事项

- **托管 / 手动互斥**:托管中入口 tab 锁,不能手动派遣 / 制造。config `auto_manage_enabled` 决定走哪条。
- **满级前提**:脚本按满级(复兴度 45)设计;未满级时部分子玩法 / 邦布数量不足可能跑不全。
- **邦巢快速选择**:脚本购买邦布需关「快速选择」(否则误买)。

## 5. 使用前置

- **游历**:至少 3×4×2=24 只邦布,方便轮换。
- **制造**:至少 8 只邦布,方便轮换。
- **售卖**:至少 2 只邦布,方便轮换。
- **邦巢**:需自行手动购买一次,勾选购买确认弹窗的「不再提醒」(脚本后续购买不再弹确认)。
